# Telegram Sticker to PNG Converter Bot

Этот Telegram-бот предназначен для конвертации стикеров, отправленных пользователем, в формат PNG и отправки их обратно. Идеально подходит для тех, кому нужны стикеры в виде файлов изображений с прозрачным фоном.

## Особенности

*   Конвертация стандартных Telegram-стикеров (`.webp`) в `.png`.
*   Обработка команд `/start` и `/help`.
*   Уведомление пользователя при отправке не-стикера или анимированного стикера.

## Локальная установка и запуск

Для запуска бота на вашем локальном компьютере выполните следующие шаги:

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/IliaBebebe/telegram-sticker-bot.git
    cd telegram-sticker-bot
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv venv
    # Для Windows:
    .\venv\Scripts\activate
    # Для macOS/Linux:
    source venv/bin/activate
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Настройте переменные окружения:**
    Создайте файл `.env` в корневой директории проекта со следующим содержимым:
    ```
    TELEGRAM_TOKEN=ВАШ_ТОКЕН_БОТА
    WEBHOOK_URL=https://your-ngrok-url.ngrok.io # Или любой другой URL для локального тестирования
    ```
    Вы можете получить `ВАШ_ТОКЕН_БОТА` от @BotFather в Telegram. `WEBHOOK_URL` нужен, если вы хотите локально тестировать вебхуки, например, с помощью ngrok. В противном случае, для локального запуска без вебхуков можно оставить его пустым или использовать заглушку.

5.  **Запустите бота:**
    ```bash
    python main.py
    ```
    *Примечание: Локальный запуск `main.py` сейчас настроен для режима отладки и может не работать с вебхуками без дополнительной настройки (например, ngrok). Для полноценного тестирования вебхуков рекомендуется развернуть бота на хостинге, например, Render.*

## Развертывание на Render

Для развертывания на Render следуйте инструкциям на [Render.com](https://render.com/). Убедитесь, что вы настроили переменные окружения `TELEGRAM_TOKEN` и `WEBHOOK_URL` в настройках сервиса Render, а также используете `waitress-serve --host=0.0.0.0 --port=$PORT wsgi:application` как команду запуска.
